# System Architecture & Design Decisions

## Overview

ESP32 Talking Agent is a self-contained voice interface platform. This document covers the key architectural choices made during design.

---

## System Architecture

The system is split into five functional subsystems:

1. **MCU & Compute** — ESP32-S3
2. **Audio I/O** — Stereo mics + stereo amps
3. **Power Management** — USB-C input, LiPo, fuel gauge, load switches
4. **User Interface** — TFT display + IMU gesture input
5. **Expansion** — STEMMA QT / Qwiic I²C connectors

---

## MCU: ESP32-S3-WROOM-1-N18R8

**Why ESP32-S3 over ESP32?**
- Native USB (USB OTG) — no USB-to-UART bridge chip needed
- 8MB PSRAM available in WROOM module — necessary for audio buffering
- Hardware acceleration for neural network inference (useful for wake word detection)
- More GPIO than base ESP32

**Why N18R8 variant?**
- 18MB Flash — room for firmware + assets (fonts, images, audio samples)
- 8MB PSRAM — sufficient for audio ring buffers and frame processing

## Audio Subsystem

### Microphones: ICS-43434 (×2)

- Digital I²S output — no ADC noise floor issues
- Flat frequency response across voice band (20Hz–20kHz)
- L/R channel select via SD pin — allows true stereo with one I²S bus
- Low power — can be power-gated via MIC_EN when not recording

### Amplifiers: MAX98357A (×2)

- Direct I²S input — no DAC chip needed
- Class-D — high efficiency, low heat
- 3W output at 4Ω — adequate for small enclosure speakers
- No I²C configuration required — GAIN set via resistor
- SPK_EN line allows power gating when audio not playing

### EMI Filtering

Ferrite beads (YI201209U121-5R0T) are placed on speaker output lines to suppress high-frequency switching noise generated by the Class-D amplifiers. This prevents interference with the microphones and RF subsystem on the ESP32.

## Power Management

### Architecture
```
USB-C (VBUS 5V)
    │
    ├──► MCP73831T ──► LiPo Battery (VBAT)
    │                        │
    └────────────────────────┤
                             │
                        DMG3415U (P-Ch MOSFET)
                        Power path selector
                             │
                           VBAT rail
                             │
                     3.3V Regulator
                             │
                           3V3 rail
```

**Why P-channel MOSFET for power path?**
Allows automatic switchover between USB and battery without an ideal diode controller IC, reducing BOM cost.

### Load Switches (TPS22918)

Three TPS22918 switches provide independent power gating for:
- Display (TFT_EN)
- Microphones (MIC_EN)
- Amplifiers (SPK_EN implied via MAX98357A SD pin)

This allows each subsystem to be fully de-powered when not in use, significantly reducing idle current.

### Fuel Gauge: MAX17048

- I²C interface — easy to poll from ESP32
- ModelGauge algorithm — accurate SOC without calibration
- ALRT pin connected to ESP32 for low-battery interrupt

---

## Display Subsystem

- SPI interface — fast enough for UI animations, uses fewer GPIO than parallel
- PWM backlight via AO3400A N-channel MOSFET controlled by ESP32 (TFT_DIM)
- TFT_EN load switch allows full display power-off

---

## IMU: LSM6DSOXTR

- I²C interface (shares bus with fuel gauge and STEMMA connectors)
- Two interrupt lines (IMU_INT1, IMU_INT2) for hardware wake and gesture detection
- Machine learning core built in — can run simple gesture classifier on-chip
- Ultra-low power in sleep mode — suitable for always-on wake detection

---

## I²C Bus

All I²C devices share a single bus:

| Device       | Address | Purpose          |
|--------------|---------|------------------|
| MAX17048     | 0x36    | Fuel gauge       |
| LSM6DSOXTR   | 0x6A/6B | IMU              |
| STEMMA QT ×2 | User    | Expansion        |

Pull-up resistors: 10kΩ to 3.3V (dedicated I²C pull-up sub-circuit)

---

## Known Limitations & V2 Improvements

- GPIO numbers not finalized in schematic — need to verify no conflicts with ESP32-S3 strapping pins
- No ESD protection on I²C lines to STEMMA connectors
- Single-sided power path — consider dedicated power path controller in v2 (e.g. BQ24075) for more robust USB/battery management
- GAIN resistors on MAX98357A should be tuned based on actual speaker impedance and desired SPL
- Consider adding a hardware mute button for mic privacy
